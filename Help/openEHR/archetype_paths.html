Archetype paths

Archetype paths are a feature of the openEHR archetype system and enable data points to be identified. Further, the are related to XPath statements, so if the data is stored as XML it is easier to build tools to access the data.
Paths have a particular feature, similar to paths used in file systems, that require the 'run time' path to be unique. This means that every piece of data located within a Composition (and for that matter within an EHR) has a unique path. This enables referencing of all data using run time paths, and all data points using the archetype path.

Paths within a particular archetype

When you build an archetype, you are creating the paths without realising it - it is a built in feature of the archetype tools. To view the paths you can:
  • Use the Archetype Workbench available from openEHR and Ocean Informatics
  • Go to the Terminology > Term Bindings Tab and move the cursor over this prepresentation of the archetype.

Run time paths

A run time path is created at run time - that is to say it only exists in the data itself. If a data point has a maximum occurrence of 1, then the archetype path is sufficient to locate the information (as no matter what the labelling - it can be identified from the archetype node ID which is stored with the data. However, if there are more than one occurrence, then the run time name will identify the data instance. This means that all run time labels (which the user sees) have to be different if they occur at the same point.

Example

In microbiology - the antibiotic to which the sensitivities relate may be repeated as many times as required:
    Penicillin - Sensitive
    Amoxycillin - Sensitive
    Trimethoprim - Resistant
The archetype path has a place holder (atXXXX code) for the antibiotic, but the data is identified by the actual label used at run time.

In a query where you wanted to know all the antibiotic sensitivities - you would use the archetype path. To recover information about sensitivities to a particular antibiotic, you would need to use the run time path which would include the representation of 'Penicillin'.
