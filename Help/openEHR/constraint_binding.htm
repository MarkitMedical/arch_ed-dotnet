Constraint definitions and bindings

Archetypes have a close relationship with terminologies - but there is no primacy of any particular terminology in the archetype system.

Constraint definitions are text - in each available language - which describe (loosely) the sort of terms that would be appropriate at this data point. An 'acXXXX' code is generated in the archetype definition as a surrogate for this statement.

This definition can then be bound to any terminology - by a constraint definition. There is, at the time of writing, no standardised way of describing a set of terms within a terminology - but this is changing. A constraint binding is a statement that will return the set of legal terms or phrases that can be added to the data at this point in the archetype.
