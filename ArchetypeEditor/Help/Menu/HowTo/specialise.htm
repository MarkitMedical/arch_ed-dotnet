Specialise an archetype

The advantage of specialisation is that data based on the specialised archetype may be returned in a query based on the parent archetype. Examples of this are:

  • diagnosis is a specialisation of problem, so all diagnoses will be returned when seeking all the problems from which a person is or has suffered
  • birth weight is a specialisation of weight, so birth weights will be returned in a query seeking all recorded weights

The converse is not true i.e. a search for all birth weights will not find the weights.

To specialise an archetype, choose Menu > File > Specialise and confirm that you wish to proceed.

You will then be asked to enter a word or very brief phrase to identify the new archetype.

In the examples above - 'diagnosis' and 'birth' would be appropriate giving the archetype names 'problem-diagnosis' and 'weight-birth' respectively.

The archetype is now specialised and it is possible to specialise an element within the archetype and add new elements and clusters. It is not possible to delete an element introduced at the level above the specialisation unless it is optional.

You will notice when you open an archetype that has been specialised that the specialisation hierarchy is shown in the header.
