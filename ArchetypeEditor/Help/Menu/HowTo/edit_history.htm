Edit the 'history' constraints of an archetype

The HISTORY class of the openEHR model enables formal representation of time and its relation to captured data. This class is described in detail in the openEHR documentation.

The history model is only available when the OBSERVATION component is chosen when creating a new archetype.

Every observation has an 'any event' event - seen in the screen shot below.

Constraints that can be set on the 'history' are:

  • Set the list of events to 'fixed' meaning that no new events can be added in specialisations, only those in the list can be used.
  • Set the events to a regular time period - which involves setting this time period.

Constraints which can be set on any event are:

  • Set the minimum and maximum occurrences of this event in the data - unbounded means there is no upper limit
  • Set the description of the event by typing in the description field
  • Name the run-time constraint
  • Set the name of the event

Events are either:

  • Point in time - relate to a particular moment.
  • Interval - relate to a period of time.

Each of these have their own set of constraints
