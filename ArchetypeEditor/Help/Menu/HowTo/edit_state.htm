Edit the 'state' constraints of an archetype

To edit the State data of an entry:
  • Select the Entry model Tab.
  • If state data is present  - the 'Person state' check box will be ticked - select the 'Person state' Tab. If not, tick the person state check box.

  • Now proceed as for other archetyped data
NB: The Archetype Editor and ADL will allow a specific history model to be developed for state data - the Person State with History option. It is not recommended that this is used at this time as processing implications of data in this format have not been tried or tested.
