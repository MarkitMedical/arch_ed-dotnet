SECTION class

In the openEHR reference model a 'composition' or document contains one or more organising 'sections' - which themselves contain 'entries'. The SECTION class of the openEHR reference model is an organising 'container'. The purpose of a section is to bring together information in a way that is useful for clinicians - it provides the 'clinical interoperability'. Importantly, a SECTION does not alter the meaning of any of the ENTRY classes held within it - this is a crucial principle when working with the openEHR architecture - and enables information to be organised in the manner that is suitable for the primary purpose it is collected - while  ensuring that it can safely be re-used for secondary purposes.

SECTION classes consist of:

  • Further SECTION classes; or
  • Slots which allow some or all ENTRY classes or SECTION classes to be contained at this point at run-time.

Section classes can be archetyped; that is to say, you can create an archetype that states the 'constraints' about what sections are contained within a particular section, the cardinality of these sections as well as what sections and entries may be added beneath the section at run-time.
