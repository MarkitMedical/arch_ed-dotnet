When to use a COMPOSITION

A COMPOSITION is a container class which has is committed to the EHR. It is used to represent documents or information which is required to be committed to the EHR as a unit. Any information which is to be collected and changed independent of other information in the EHR requires its own composition.

Examples of Compositions are:

  • Documents created by a clinician and stored in the EHR
  • Laboratory results and report
  • ECG results and report
  • A Problem list which is separate from any particular document
  • A Family History list which is separate from any particular document.

In summary, if you are trying to standardise information that is usually committed as a 'document' or at least by a clinician at one time, then you probably need to use a COMPOSITION.
