When to use an EXTRACT

An EXTRACT is a collection of COMPOSITIONs which can be transferred from one EHR to another. These compositions may already exist in the 'sending' EHR or may be created especially for the communication. If you are trying to represent a communication between clinicians - from one EHR to another EHR - it will always involve an extract. An archetype is not required to create an Extract, rather to set what the Extract might or must contain.

Examples of Extracts are:

  • Discharge summary
  • Referral communication
  • Health summaries communicated to a central organisation

In summary, if you are attempting to standardise or constrain complex and variable communication between clinicians, involving information already held in the EHR - then you probably need to create an archetype for the EXTRACT class.
