Set constraints on an element or cluster

All elements and clusters have common constraints - which are present on the top left hand panel.

Occurrences

The first numeric counter from the left sets the minimum number of occurrences of this element or cluster. The default is 0, which indicates that this element or cluster is optional. The maximum number of occurrences can be set in the numeric box - the default is 1 - or by ticking the unbounded box to indicate there is no upper limit. It is important to realise that if a node's maximum occurrences are set to a number greater than 1 (or unbounded), then the data points will be identified by the 'meaning' attribute in the openEHR reference model. The editor calls this attribute the run-time name - and enables you to set a 'title' for the constraint on these runtime names - called the runtime name constraint.

Description

The description adds clarity to other users of the archetype when identifying the data point represented.

Runtime name constraint

At present there is no standard means of constraining a set of terms from an external terminology. If you add text to the Runtime name constraint, a code acXXXX will be created which can then be bound to an external terminology. The text at this point in the editor should give some indication of the type of constraint intended.

Further constraints depend on the data type:

  • Text
  • Quantity
  • Count
  • Ordinal
  • Date/Time
  • Boolean
  • Any - no constraints
