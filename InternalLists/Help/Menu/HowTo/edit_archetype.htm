Edit an existing archetype

The archetype editor is distributed with a small sample set of archetypes; this help uses the blood pressure archetype which has a file name 'openEHR-EHR-OBSERVATION.blood_pressure.v1.adl' as an example. It can be found in the Archetype directory below where the Archetype Editor is installed (c:\Program Files\Ocean\Archetype Editor) is the default. You can download a larger set of archetypes from openEHR.

An existing archetype can be loaded from the Start Up screen when starting the editor, or when the editor is already running from the 'File > Open' Menu - or the short-cut key {Ctrl-O}.

There are a range of actions the user may undertake to update an archetype. These include:

  • Edit the header - edit the concept or description, restrict the subjects of care etc.
  • Edit the data constraints - edit the core data associated with an archetype.
  • Edit or add state constraints - information which may be collected at the same time as the core data which may alter the interpretation of the information.
  • Edit or add protocol constraints - information about the data which is not required for interpretation.
  • Edit or add history constraints - whether an observation relates to a point in time or relates to a period of time.
  • Translate an archetype - which entails adding bindings to another language.
  • Add a terminology - add a new terminology to enable bindings to particular concepts represented within it.
  • Add bindings to a terminology - add a binding of a concept represented within the archetype to the terminology.
  • Specialise an archetype or element - enabling a more specialised concept to be represented as a 'child' of the parent concept.
